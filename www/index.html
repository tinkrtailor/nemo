Deploy a nautiloop

The nautiloop Terraform module installs on any Linux server with SSH access. You provision the server, the module handles k3s, Postgres, and the control plane. Self-contained: no extra provider configuration needed. Single terraform apply from a clean state.

main.tf 
module "nautiloop" {
  source = "github.com/tinkrtailor/nemo//terraform/modules/nautiloop"

  server_ip       = "100.64.0.1"                          # any server with SSH
  ssh_private_key = file("~/.ssh/id_ed25519")
  git_repo_url    = "git@github.com:you/your-repo.git"
  git_host_token  = var.github_pat
}

Four required variables. terraform apply. Done. Full deploy guide →

Security model

Agent containers get open internet but no direct access to secrets. Credentials are injected by a localhost sidecar.

  • Auth sidecar proxies model API calls and git pushes, injecting credentials without exposing them as env vars or files
  • Read-only reviewer mounts the worktree read-only in review stage
  • Shared API key auth (V1) — designed for single-tenant / small-team deployments
  • Egress logging on all outbound traffic from agent pods
  • Tailscale recommended for private API access
